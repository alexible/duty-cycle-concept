🔄 What is a Complete Cycle?

Complete Cycle: One full repetition of a pattern - from start, through all changes, back to start

🌊 Real-World Examples

• Clock tick: Tick-tock-tick = 1 cycle

• Heart beat: Lub-dub-pause = 1 cycle

• Breathing: Inhale-exhale = 1 cycle

⚡ Electronic Cycles

• AC power: 50Hz = 50 cycles/second

• PWM signal: 1kHz = 1000 cycles/second

• Each cycle: ON time + OFF time

💡 Key Understanding:

A cycle repeats over and over. Duty cycle tells us what percentage of each cycle the signal is ON.

📖 What is Duty Cycle?

Duty Cycle: The percentage of time a signal is in its "high" or "on" state during one complete cycle

Key Concepts:

• Period (T): Total time for one complete cycle

• On-Time (t_on): Time signal is HIGH/ON

• Off-Time (t_off): Time signal is LOW/OFF

• Relationship: T = t_on + t_off

📐 Duty Cycle Formula

Duty Cycle (%) = (t_on / T) × 100%

Where: t_on = ON time, T = Total period

🤔 Why Do We Need Duty Cycle?

🎯 The Problem: Why Not Just Use a Potentiometer?

🔧 Potentiometer Method

How: Variable resistor reduces voltage

Problem: Wastes energy as heat

Example: 50% dimming = 50% energy lost as heat!

Issues: Hot, inefficient, limited control

⚡ PWM/Duty Cycle Method

How: Rapidly switches ON/OFF

Advantage: No energy wasted as heat

Example: 50% dimming = 50% energy to LED

Benefits: Cool, efficient, precise control

🏆 Key Benefits of Duty Cycle Control:

1. Energy Efficiency: Up to 95% efficient vs 50% with potentiometer

2. Precise Control: Can adjust in tiny increments (0.1%)

3. No Heat Generation: Switch is either fully ON or OFF

4. Digital Control: Easy to control with computers/microcontrollers

5. Fast Response: Can change thousands of times per second

💡 Real Example:

Your phone's screen brightness uses PWM - that's why it's so efficient and doesn't get hot!

🎛️ Who Controls the ON/OFF Switching?

🔌 The Electronic "Switch" - Who's in Control?

🧠 The Controller

Microcontroller: Arduino, Raspberry Pi

IC Chips: Timer ICs (555 timer)

Dedicated PWM: Motor driver chips

Computer: Software-generated signals

⚡ The Physical Switch

MOSFET: Most common (electronic switch)

Transistor: BJT for smaller loads

Relay: For high-power AC loads

Integrated: Built into motor drivers

🔄 The Process:

1. Controller decides when to switch (duty cycle)
2. Electronic switch (MOSFET) turns power ON/OFF
3. Load (LED/motor) receives chopped power
4. Process repeats thousands of times per second

⚡ How PWM Controls Power Delivery

🔋 The Magic: Average Power Control

🧮 The Math

Full Power: 12V × 100ms = 1200V⋅ms

25% Duty: 12V × 25ms = 300V⋅ms

Result: 25% of full power delivered

💡 LED Example

100% Duty: Maximum brightness

50% Duty: Half brightness

10% Duty: Very dim

Eye sees: Average brightness

🔧 Motor Example

100% Duty: Full speed

75% Duty: 75% speed

25% Duty: Slow speed

Motor feels: Average voltage

🎯 Key Insight:

PWM tricks the load into "thinking" it's getting a lower voltage by rapidly switching full voltage ON and OFF!

⚡ PWM (Pulse Width Modulation)

PWM: A technique that uses duty cycle to control power delivery

How PWM Works:

1. Fixed frequency (period stays constant)

2. Variable pulse width (duty cycle changes)

3. Average power varies with duty cycle

💡 Result:

25% duty cycle = 25% average power = Dim LED

75% duty cycle = 75% average power = Bright LED

🤖 Why Servos Work ONLY on Duty Cycle

🎯 Servo Design: Built for Pulse Width Communication

🧠 Inside a Servo

• Control circuit: Reads pulse width

• Potentiometer: Senses current position

• DC motor: Moves to target position

• Feedback loop: Compares pulse vs position

⚡ Why PWM/Duty Cycle?

• Digital communication: Immune to noise

• Precise timing: Microsecond accuracy

• Standard protocol: Works with any controller

• Simple wiring: Just 3 wires needed

🏆 Benefits of Servo PWM Control:

1. Absolute Positioning: Pulse width = exact angle

2. No Drift: Servo holds position without constant commands

3. Noise Immunity: Digital signal unaffected by electrical noise

4. Universal Standard: Same protocol for all servo brands

5. Simple Interface: One signal wire controls everything

💡 Why Not Voltage Control?

Problem: Voltage levels vary, drift over time, affected by noise
Solution: Time-based control is precise, digital, and reliable!

🤖 Arduino PWM Example

Practical Implementation: LED Brightness Control

// Arduino PWM Example - LED Brightness Control
int ledPin = 9;      // PWM pin
int brightness = 0;  // PWM value (0-255)

void setup() {
    pinMode(ledPin, OUTPUT);
}

void loop() {
    // Fade in
    for (brightness = 0; brightness <= 255; brightness += 5) {
        analogWrite(ledPin, brightness);  // Set PWM duty cycle
        delay(30);
    }
    
    // Fade out
    for (brightness = 255; brightness >= 0; brightness -= 5) {
        analogWrite(ledPin, brightness);  // Set PWM duty cycle
        delay(30);
    }
}

🔢 PWM Values

0: 0% duty cycle (LED OFF)

127: 50% duty cycle (Half brightness)

255: 100% duty cycle (Full brightness)

⚙️ How it Works

analogWrite(): Generates PWM signal

Frequency: ~490 Hz (Arduino Uno)

Result: Smooth LED brightness control

🦾 Servo Motor Control

Servo Control: Uses duty cycle to set precise angular position

How Servo PWM Works:

• Frequency: 50 Hz (20ms period)

• Pulse Width: 1-2ms determines angle

• Duty Cycle: 5-10% range

Pulse Width Duty Cycle Servo Position Angle
1.0 ms 5% Minimum 0°
1.5 ms 7.5% Center 90°
2.0 ms 10% Maximum 180°

🎯 Key Point:

Small changes in duty cycle create precise angular control

This is why servos are perfect for robotics and RC vehicles!
